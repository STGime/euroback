Beta update · June 2026

Hi,

We spent the last days building a real app on Eurobase ourselves — and fixed everything that got in the way. The result is a big batch of improvements, most of them to edge functions. Everything below is already live; no action needed on your side.

⚡ Edge functions

TypeScript & export default now work

Write your function in TypeScript with a standard export default async (req, ctx) => … — we compile it when you deploy. Broken code now fails at deploy time with line numbers instead of erroring on first invocation. One honest limitation: third-party imports (URL or npm) aren't supported yet — using one returns a clear runtime error rather than failing silently.

Vault secrets work inside functions

ctx.vault.get('MY_API_KEY') previously returned null for recently written secrets — fixed. And when something is wrong on our side, you now get a descriptive error instead of a null that looks like a missing secret.

Row-level security behaves like the REST API

auth_uid() and is_service_role() now work in SQL run from functions: a function invoked with a user JWT acts as that user, and without one it gets service access — exactly like the REST API. If you wrote RLS policy workarounds for functions, you can delete them.

Redeploys take effect immediately

Previously a redeployed function could keep running old code for up to 5 minutes. Now a deploy is live on the very next invocation.

Schedules accept real timezones

timezone: 'Europe/Berlin' (or any IANA timezone) now works for scheduled functions — no more UTC-only workarounds.

🛠 CLI & getting started

Install the CLI with Homebrew

The CLI is now properly distributed — install or update it with:

brew install stgime/tap/eurobase

Check what you're running with eurobase version — future updates arrive via brew upgrade. Prefer a plain binary? Download it here.

The CLI tells you where things go

eurobase status now actually shows your active project, plan, and usage. Deploys name their target ("deployed to predwell (0515e4e2…)"), and login can no longer leave you silently pointed at an old project — so a deploy can't land somewhere you didn't expect.

localhost works out of the box

New projects now allow browser requests from http://localhost:3000 by default — no more CORS wall on your first fetch. For existing projects, add your app's origin under Project → Auth settings → CORS origins.
